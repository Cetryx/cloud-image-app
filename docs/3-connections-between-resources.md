# 3. Connections between resources

The resources are connected through Terraform references, which ensures that Terraform creates them in the correct order.

The Resource Group is the base resource. The Storage Account and Key Vault both reference the Resource Group name and location:

- `azurerm_storage_account.main` uses `azurerm_resource_group.main.name`
- `azurerm_storage_account.main` uses `azurerm_resource_group.main.location`
- `azurerm_key_vault.main` uses `azurerm_resource_group.main.name`
- `azurerm_key_vault.main` uses `azurerm_resource_group.main.location`

The Blob containers depend on the Storage Account because they use `azurerm_storage_account.main.id`:

- `azurerm_storage_container.images`
- `azurerm_storage_container.tfstate`

The Key Vault depends on the authenticated Azure tenant context because it uses `data.azurerm_client_config.current.tenant_id`.

The RBAC assignments connect the currently authenticated Azure user or service principal to the created resources:

- `current_user_storage_blob_data_contributor` assigns `Storage Blob Data Contributor` on the Storage Account.
- `current_user_key_vault_secrets_officer` assigns `Key Vault Secrets Officer` on the Key Vault.

The random suffix generated by `random_string.suffix` is used by the local names for globally unique resources:

- Storage Account: `stcloudimg<variant><suffix>`
- Key Vault: `kv-cloud-img-<variant>-<suffix>`
- Web App name prepared in locals: `app-cloud-img-<variant>-<suffix>`
